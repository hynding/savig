# Scissors Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scissors tool (`c`): click a segment — closed paths open at the point, open paths split into two objects; geometry-exact (de Casteljau) and position-exact (absolute-anchor pinning).

**Architecture:** Engine `cutPath(path, segmentIndex, t)` built on `splitCubicRange`/`evalCubic` (boolean-curves). Store `cutSelectedPathAt(segmentIndex, t)` with gates (non-path/morph/rings/boolean → toast no-op), one-commit split creating piece b via the addVectorPath/booleanOp precedents, trim+dashOffsetTrack dropped, anchors pinned absolute. Stage: node-branch-style ring-scan hit-test with chord-t→curve-t re-projection; tool stays active across cuts.

**Tech Stack:** pnpm monorepo, TS strict, Vitest colocated, Playwright e2e.

**Spec:** `docs/superpowers/specs/2026-07-10-scissors-design.md` (approved). Anchors from a verified seam survey; re-locate by pattern if drifted.

## Global Constraints

- Cut anchors are corner nodes carrying the EXACT subdivided handle pairs; pieces must have ≥2 nodes (else `{kind:'noop'}`).
- Position exactness: `split` pieces keep nodes in the ORIGINAL local coords, `base` copied verbatim, and BOTH pieces get `anchorMode:'absolute'` at the original object's pre-cut resolved anchor point (local coords). `opened` keeps the object's anchor untouched.
- Dropped on cut pieces: `trim`, `dashOffsetTrack` (path re-parameterization). Copied verbatim: transform `tracks`, `motionPath`, `repeat`, style (deep copy for piece b's new asset). Style `strokeDasharray` kept.
- Gates (toast + no commit): non-vector-path target; `shapeTrack.length > 0`; `compoundRings` non-empty; `obj.boolean` present.
- ONE commit per cut (single undo restores fully). Selection after split: `selectedObjectIds:[aId,bId]`, `selectedObjectId: bId`.
- Scissors STAYS active after a cut (unlike eyedropper). Empty-canvas click: no-op.
- `packages/engine/src/cutPath.ts` will be in the runtime bundle graph ONLY if something in the runtime imports it — it won't be (editor-only op); do NOT export it through any module the runtime pulls unless unavoidable; verify with a grep and skip bundle regen if unreferenced. State the verdict in the Task 1 report.
- Env: `node_modules/.bin/{vitest,tsc,eslint,playwright}` from repo root; NEVER `pnpm install`/`pnpm approve-builds`; revert stray `pnpm-workspace.yaml` after e2e. Fresh `useEditor.getState()` per test read.
