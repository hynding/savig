# Blend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Blend command — exactly 2 selected vector paths generate n intermediate objects interpolating world-space geometry, paint, strokeWidth, and opacity, with easing.

**Architecture:** Pure engine `computeBlendSteps(project, objA, objB, {count, easing?, time?}): BlendStep[] | null` (computeOutlineStrokeEffect dual-consumer precedent) reusing `reconcile`/`resample`/`align`/`suggestCorrespondence` on WORLD-space paths (handles via the affine offset identity). Store `blendSelected(count, easing?)` loops steps → assets+objects, ONE commit (duplicateSelected accumulate + cutSelectedPathAt two-step scope/assets composition). `canBlend` predicate + `path.blend` command + Inspector exactly-2 panel. DSL `blendPaths` + MCP `blend`.

**Tech Stack:** pnpm monorepo, TS strict, Vitest colocated, Playwright e2e.

**Spec:** `docs/superpowers/specs/2026-07-11-blend-design.md` (approved; Decisions 1–11 binding).

## Global Constraints

- **No model changes** — intermediates are plain vector `path` assets + objects, `primitive: undefined`, no tracks, no copied dash/trim/textPath.
- **World-space interpolation:** sample A/B at the playhead (`sampleObject`), map nodes through the FULL parent-chain transform (anchors `mapPoint`; in/out handle offsets via `world(anchor+offset) − world(anchor)`); intermediates bake identity rotation/scale, normalized to local-origin bbox + `anchorMode: 'fraction'` (applyBooleanResult precedent).
- **Correspondence:** equal node counts → `reconcile(aW, bW, 'corresponded', suggestCorrespondence(aW, bW))`; unequal → `reconcile(aW, bW, 'resampled')`. `closed` held from A. t_i = `applyEasing(easing ?? 'linear', i/(n+1))`, endpoints excluded.
- **Style:** solid↔solid `interpolateColor`; gradient↔gradient `interpolateGradient`; ANY paint-kind mismatch STEPS holding A until t ≥ 1. strokeWidth + opacity numeric lerp (opacity → `base.opacity`).
- **Eligibility (`isBlendEligible`, checked per object; `canBlend` = exactly 2 pass): `isLockedInTree` FIRST (mutating-action rule), then: vector asset, `shapeType === 'path'`, non-empty `asset.path`, no `compoundRings`, no `shapeTrack`, no `repeat`, not `obj.boolean` result, not a boolean operand, not group/instance/svg/text.** Grouped leaves allowed; intermediates land at scope ROOT. A = lower zOrder, B = higher.
- **One commit**; created ids selected after; names `Blend 1..n`; zOrder via `nextZOrder`, A→B order.
- blend.ts must NEVER be imported from `packages/runtime/src` (editor-only, tree-shaken). Exporting `lerpNode`/`lerpPoint` from path.ts touches the runtime bundle graph → regen `(cd packages/runtime && node scripts/build-runtime.mjs)` in the SAME commit + regen-and-diff verdict.
- Env: `node_modules/.bin/{vitest,tsc,eslint,playwright}` from repo root; NEVER `pnpm install`/`pnpm approve-builds`; revert stray `pnpm-workspace.yaml`. Fresh `useEditor.getState()` per test read.
- Commits end with `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
