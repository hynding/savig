# Repeater Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `repeat: { count, dx, dy, rotate, scale, stagger }` on leaf SceneObjects renders N transformed, time-staggered copies everywhere.

**Architecture:** Expansion lives at the ONE walker seam — `flattenInstances`' plain-leaf branch emits `count` leaves (`renderId` bare for k=0, `@k` suffix for k≥1; per-copy transform delta composed into `transformPrefix`; `localTime − k·stagger` clamped ≥0). Static markup, runtime frames, raster, and both apps inherit it by construction. Editor: copies hit-route to the source; Inspector panel mirrors the Symbol-timing block; agent surface via core builder + DSL field + MCP tool.

**Tech Stack:** pnpm monorepo, TS strict, Vitest colocated, Playwright e2e at repo-root `e2e/`.

**Spec:** `docs/superpowers/specs/2026-07-10-repeater-design.md` (approved). File:line anchors from a verified seam survey — re-locate by pattern if drifted.

## Global Constraints

- **Parity:** absent `repeat` → byte-identical leaves/renders. With `repeat`, the k=0 leaf is byte-identical to the unrepeated leaf (bare renderId, unchanged transformPrefix, unchanged localTime).
- `count` static, integer, clamped [2, 64] when active; `count ≤ 1` normalizes `repeat` to `undefined` (`normalizeRepeat`). `stagger ≥ 0` finite; `dx/dy/rotate` finite; `scale` clamped [0.01, 100]. Rotate is DEGREES.
- v1 scope: repeat on plain leaf objects only (vector/text/svg-asset) — `canRepeat` = non-group, non-instance. A repeated leaf INSIDE a symbol is allowed (walker recursion handles it; ids `inst/child@k`).
- Per-copy delta composes AFTER the ambient prefix, BEFORE the object's own transform: `translate(k·dx, k·dy) rotate(k·rotate) scale(scaleᵏ)`; all numbers through `fmt()`; k=0 emits no delta string at all.
- `packages/engine/src/symbol.ts` is in the runtime bundle graph — any commit touching it regenerates the bundle IN THE SAME COMMIT: `(cd packages/runtime && node scripts/build-runtime.mjs)`; `git add packages/runtime/src/runtimeSource.generated.ts`.
- Store ops via `selectActiveObjects`/`selectActiveScope`; sparse-omit + byte-clean JSON (`repeat: undefined` clears, never null/defaults).
- Test gotcha: fresh `useEditor.getState()` per read.
- Env: `node_modules/.bin/{vitest,tsc,eslint,playwright}` from repo root; NEVER `pnpm install`/`pnpm approve-builds`; revert stray `pnpm-workspace.yaml` after e2e.
